# SPSU26 Linesheet.pdf 解析与飞书知识库方案

## 文档基本情况
- 文件：`/Users/miaozixin/Documents/EZR/项目/PVH/CK商品数据/SPSU26 Linesheet.pdf`
- 页数：109 页
- 类型：品牌商品资料 / linesheet / 商品培训资料

这类 PDF 很适合做两层沉淀：
1. **商品知识库**：把每个款的结构化商品信息拆出来
2. **搭配知识库**：把系列/造型/风格信息拆出来，形成可用于推荐的 look 知识

---

# 一、先说结论：怎么建最合适

## 推荐载体：全部先放飞书多维表格
原因：
- 后续要检索、筛选、做“在库约束推荐”
- 需要关联商品、库存、搭配 look
- 后续还要接导购上传图片和门店库存表

## 先建 4 张表
1. `商品知识库`
2. `搭配知识库`
3. `门店模拟库存库`
4. `搭配候选记录`

这样第一版就能跑起来。

---

# 二、PDF 解析目标

## 目标 1：从 PDF 中拆出“商品卡”
每个商品至少抽这些字段：
- 品牌
- 季节
- 系列
- 款号 / style code
- 商品名
- 品类
- 性别线（若有）
- 颜色
- 材质
- 版型
- 风格标签
- 场景标签
- 适合身形标签
- 卖点摘要
- 原始页码
- 来源文件名

## 目标 2：从 PDF 中拆出“搭配卡 / look 卡”
如果 PDF 中有系列图、搭配图、主题页、look 页，则抽这些字段：
- look_id
- 系列 / 主题
- 风格标签
- 主单品
- 搭配单品
- 色彩关系
- 层次结构
- 适合场景
- 适合身形
- 搭配逻辑
- 关联商品款号
- 原始页码
- 来源文件名

## 目标 3：建立“门店模拟库存”
第一版先模拟，不等真实库存表：
- 从商品知识库中抽一部分商品
- 给每个商品补上门店、颜色、尺码、库存数量、在库状态
- 作为推荐引擎的可卖商品池

# 三、飞书多维表格表设计

## 表 1：商品知识库
用途：承接 PDF 商品解析结果，是所有推荐的底层商品事实表。

### 建议字段
- `商品ID`：文本，主键，建议 `brand-season-stylecode`
- `品牌`：单行文本
- `季节`：单选 / 文本
- `系列`：单行文本
- `款号`：单行文本
- `商品名`：单行文本
- `品类`：单选
- `性别线`：单选
- `颜色`：多选 / 文本
- `材质`：多选 / 文本
- `版型`：单选 / 文本
- `长度标签`：单选（短款/常规/长款）
- `风格标签`：多选
- `场景标签`：多选
- `适合身形标签`：多选
- `核心卖点`：长文本
- `价格带`：单行文本
- `图片来源`：附件 / URL
- `原始页码`：数字
- `来源文件`：单行文本
- `来源摘要`：长文本
- `是否可上架推荐`：复选框

### 风格标签建议枚举
- 通勤
- 极简
- 休闲
- 轻正式
- 法式
- 美式休闲
- 运动休闲
- 松弛感
- 干练
- 性感
- 青年感

### 身形标签建议枚举
- 小个子友好
- 高挑友好
- 微胖友好
- 梨形友好
- H 型友好
- 沙漏型友好
- 肩宽友好
- 腿长展示型

---

## 表 2：搭配知识库
用途：承接 PDF 中的系列/主题/look 搭配知识，也承接后续导购上传的穿搭图。

### 建议字段
- `搭配ID`：文本主键
- `主题/系列`：单行文本
- `look名称`：单行文本
- `主单品款号`：关联到 `商品知识库`
- `搭配单品款号列表`：关联到 `商品知识库`（多值）
- `风格标签`：多选
- `场景标签`：多选
- `适合身形标签`：多选
- `色彩关系`：单行文本
- `层次结构`：单行文本
- `搭配逻辑`：长文本
- `推荐理由`：长文本
- `来源类型`：单选（品牌资料 / 导购图片 / 人工补充）
- `来源文件`：单行文本
- `原始页码`：数字
- `来源图片`：附件 / URL
- `是否可推荐`：复选框

### 搭配逻辑建议怎么写
建议统一成一句结构：
- `主单品 + 辅助单品 + 风格目的 + 身形效果`

例如：
- “短款上衣 + 高腰直筒下装，强调腰线，适合小个子显高”
- “廓形外套 + 修身内搭，形成松紧对比，适合通勤利落风格”

---

## 表 3：门店模拟库存库
用途：第一版先模拟库存，先让推荐链路跑起来。

### 建议字段
- `库存ID`：文本主键
- `门店`：单选
- `商品ID`：关联到 `商品知识库`
- `款号`：lookup
- `商品名`：lookup
- `品类`：lookup
- `颜色`：单选 / 文本
- `尺码`：单选
- `库存数量`：数字
- `在库状态`：单选（有货 / 紧张 / 无货）
- `陈列优先级`：单选（高 / 中 / 低）
- `主推状态`：复选框
- `替代推荐等级`：单选（主推 / 替代 / 补充）

### 模拟规则
第一版建议：
- 先从商品知识库里挑 80~120 个商品做模拟库存
- 每个商品随机或按品类补 2~4 个尺码
- 每条库存给一个 1~8 的库存量
- 将 70% 标记为 `有货`，20% 标记为 `紧张`，10% 标记为 `无货`

---

## 表 4：搭配候选记录
用途：记录某次“输入一件商品后，系统给出的候选搭配”。

### 建议字段
- `候选记录ID`
- `触发商品ID`：关联到 `商品知识库`
- `触发来源`：单选（商品图 / 款号 / 手动检索）
- `候选look_id`：关联到 `搭配知识库`
- `推荐门店`：单选
- `推荐风格`
- `目标身形`
- `推荐搭配商品列表`：关联到 `商品知识库`
- `推荐理由`
- `库存可执行性`：单选（可直接试 / 需替代款 / 库存不足）
- `导购话术`
- `创建时间`

# 四、PDF 解析策略

## 第一层：页面分类
先把 109 页按页面类型分类：
- 商品明细页
- 系列/主题页
- look 图页
- 品牌/说明页
- 尺码/价签/辅助页

### 分类结果怎么用
- 商品明细页 → 进入 `商品知识库`
- look 图页 / 系列页 → 进入 `搭配知识库`
- 品牌说明页 → 作为辅助标签/卖点摘要来源

---

## 第二层：商品抽取
如果某页主要是商品清单，重点抽：
- 款号
- 品类
- 颜色
- 材质
- 版型
- 价格带
- 卖点词

### 标准化规则
把原始文案映射成统一标签，例如：
- relaxed fit / loose fit / oversized → `宽松`
- slim fit → `修身`
- straight → `直筒`
- cropped → `短款`
- tailored → `利落/轻正式`

这样后面推荐才稳。

---

## 第三层：搭配抽取
如果某页有整套 look，重点抽：
- 主单品是什么
- 搭配的上/下/外/配饰分别是什么
- 这套搭配整体风格是什么
- 适合什么场景
- 对什么身形友好
- 为什么成立

### 这里的关键不是“识别几个单品”
而是要生成一条 `搭配逻辑`：
- 例如“上紧下松”“上短下长”“同色系延长比例”“外松内紧形成层次”

这个字段后面价值最大。

# 五、用 PDF 同时建立“商品知识库”和“搭配知识库”的做法

## 商品知识库建立方法
PDF 中每出现一个明确商品，就落一条商品卡。

## 搭配知识库建立方法
只要页面里出现：
- 系列主题穿搭
- lookbook 风格页
- 多件商品组合关系

就落一条搭配卡。

### 注意
一个页面可能同时产出：
- 多个商品卡
n- 1 个搭配卡

这是正常的。
