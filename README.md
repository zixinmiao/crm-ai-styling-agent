# 商品搭配 Agent 能力包

## 目标
为门店导购提供一个可执行的穿搭 Agent：
1. 管理并理解门店当前有库存的商品
2. 通过**吊牌识别**与穿搭图片拆解，沉淀商品/搭配知识库
3. 在顾客试衣时，基于吊牌命中的单品、库存、风格和身形输出试衣候选
4. 支持"拍吊牌 → 精确命中商品 → 结合顾客画像输出搭配"

## 知识库设计（飞书多维表格）

### 一、商品知识库
- 品牌 / 季节 / 系列 / 款号 / 商品名 / 品类 / 颜色 / 材质 / 版型 / 风格标签 / 场景标签 / 适合身形 / 卖点 / 来源

> `款号` 是吊牌识别的唯一精确匹配字段，必须保证全库唯一或"款号+颜色"唯一。

### 二、门店库存库
- 门店 / SKU / 款号 / 商品名 / 颜色 / 尺码 / 价格 / 库存数量 / 在库状态 / 搭配角色 / 近似替代款

### 三、搭配知识库
- look_id / 风格标签 / 主单品 / 搭配单品 / 色彩关系 / 层次结构 / 适合场景 / 适合身形 / 推荐理由 / 来源图片/来源资料

### 四、品牌方法论文档（飞书文档）
- 品牌商品培训摘要 / 不同品类卖点 / 风格搭配原则 / 身形搭配方法 / 门店搭配案例

### 五、顾客画像记录
- 顾客画像ID / 身形标签 / 风格偏好 / 禁忌点 / 试衣目标 / 推荐约束 / 来源描述

### 六、吊牌识别日志
- 识别记录ID / 输入吊牌图 / 款号 / 颜色 / 尺码 / 是否命中 / 最终命中商品 / 库存状态 / 是否走人工兜底

## 商品知识库冷启动（一次性工作，不进运行时）
`SPSU26 Linesheet.pdf` 这类品牌资料作为**一次性数据源**导入：
1. 线下解析 PDF，拆解为商品知识卡
2. 批量导入飞书「商品知识库」
3. 品牌表达/卖点总结沉淀到飞书文档
4. 产物保留在 `outputs/` 作为历史记录

> PDF 解析已不再是运行时 skill，导购端不会触发 PDF 流程。

## 运行时主链路
1. 导购拍**吊牌图** / 描述顾客 / 拍穿搭图
2. `skill-styling-tag-recognize`：OCR 款号 → 按款号精确命中商品知识库 → 联查门店库存库 → 得到 `matched_product` 和 `inventory_status`
3. `skill-styling-customer-profile-parse`：提取顾客画像、身形与约束
4. `skill-styling-outfit-recommend`：读取「搭配知识库」+「门店库存库」，仅输出“知识库命中 + 库存命中”的搭配
5. `skill-styling-sales-script-generate`：生成手机端短话术
6. 推荐结果写入「搭配候选记录」

> 运行时推荐链路直接读取现成知识库；`knowledge_sync` 主要用于库存标准化、穿搭图片拆解后的写库沉淀，不作为每次推荐的前置动作。

## 吊牌识别路径
1. **款号 OCR**：识别吊牌款号，按「商品知识库.款号」精确查；同款号多颜色时用 OCR 到的颜色二次过滤
2. **库存联查**：命中商品后，用 `款号 + 颜色 + 尺码` 联查「门店库存库」
3. **人工兜底**：款号读不出、知识库未命中或无法唯一确认时，提示导购重拍或手工输入款号，**不做模糊猜测**

详见 [skills/skill-styling-tag-recognize/SKILL.md](skills/skill-styling-tag-recognize/SKILL.md)。
