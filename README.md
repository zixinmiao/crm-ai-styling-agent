# 商品搭配 Agent 能力包

## 目标
为门店导购提供一个可执行的穿搭 Agent：
1. 管理并理解门店当前有库存的商品
2. 解析品牌商品资料与穿搭图片，沉淀商品/搭配知识库
3. 在顾客试衣时，基于单品、库存、风格和身形输出试衣候选
4. 支持“拍一张商品图 -> 匹配商品候选 -> 结合顾客画像输出搭配”

建议把以下内容沉淀到飞书文档：

### 五、顾客画像记录（后续建议补到飞书多维表格）

### 六、单品识别日志（后续建议补到飞书多维表格）

## 关于 PDF 商品资料如何建立知识库
`SPSU26 Linesheet.pdf` 这类文件很适合作为“品牌商品知识”的初始来源。
建议流程：
1. 先把 PDF 按系列/款号拆解成商品知识卡
2. 将结构化字段写入飞书多维表格的 `商品知识库`
3. 将每页/每个系列的品牌表达、卖点总结沉淀到飞书文档
4. 后续再与门店库存表做映射，形成“可卖商品库”

## 回答你的问题：是否可以建立到飞书文档上？
可以，但建议分层：
- **飞书多维表格**：放结构化商品知识、库存数据、搭配知识卡
- **飞书文档**：放培训摘要、风格说明、搭配方法论、导购使用手册

不要只放飞书文档。因为搭配推荐和检索更依赖结构化数据。飞书文档更适合承载“解释”和“方法”。

## 当前推荐主链路
1. 导购上传单品图 / 款号 / 顾客描述
2. `skill-styling-item-match` 识别单品并匹配商品候选
3. `skill-styling-customer-profile-parse` 提取顾客画像、身形与约束
4. `skill-styling-outfit-recommend` 读取飞书表中的商品知识库、搭配知识库、门店模拟库存库，输出 3-5 套候选
5. `skill-styling-sales-script-generate` 生成导购可直接使用的话术
6. 推荐结果写入 `搭配候选记录`
